В данном репозитории вы можете найти инструкцию, как поднять свой собственный overleaf локально в Docker.

**Overleaf** — это популярный облачный онлайн-редактор для создания документов с использованием системы верстки LaTeX. Он позволяет писать научные статьи, диссертации и отчеты в браузере без установки программ, поддерживая совместную работу в реальном времени, контроль версий и шаблоны ведущих издательств.

Если по тем или иным причинам официальное [облачное решение](https://ru.overleaf.com/) вас не устраивает, есть open source часть, которая позволяет развернуть сервис локально.
Это может быть внутренний сервер вашей организации, ваш личный сервер или пользовательский девайс.

[Ссылка на официальную open source часть](https://github.com/moritzgloeckl/overleaf-sync) содержит весь основной код, однако при запуске в докере могут возникнуть некоторые проблемы. Для решения этих проблем я подготовил патч, который позволяет сформировать корректный образ с множеством дополнительных LaTeX пакетов, необходимых для написания научных трудов, а так же корректно развернуть итоговый сервис в docker.

## Quick Start

Необходимо скачать оригинальный код и патч overleaf.patch из текущего репозитория
```
git clone https://github.com/moritzgloeckl/overleaf-sync
cd overleaf
```

Поместите патч overleaf.patch в корень основного проекта и используйте git, чтобы применить его
```
git apply overleaf.patch
```

После чего можно будет собрать и запустить сервис в докере:
```
docker compose build --no-cache sharelatex && docker compose up -d 
```

При первом скачивании подтянется основной образ sharelatex. Скачивание и сборка кастомного образа могут занять некоторое время.
После того, как все основные службы поднимутся в докере, можно будет перейти в браузере по адресу
```
http://localhost/launchpad
```
После этого вы попадете на стандартную страницу авторизации и регистрации в своем личном overleaf.
